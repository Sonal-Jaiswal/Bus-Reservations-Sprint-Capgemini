# Bus Cleaning Pipeline (Java Only)

This is a pure Java project that reads one raw CSV file, applies all cleaning/transformation rules, logs each step in console/log file, and writes cleaned/invalid/aggregated outputs.

## Input and Output

- Input CSV (you provide): `data/raw_booking.csv`
- Cleaned CSV: `output/cleaned_bookings.csv`
- Invalid CSV with reason: `output/invalid_bookings.csv`
- Aggregated summary: `output/aggregated_report.csv`
- Execution logs: `logs/cleaning.log`

## Cleaning Rules Implemented

1. Remove duplicates (`booking_id`)
2. Normalize names (trim + proper case)
3. Validate numeric fields (`age`, `seats`, `amount`)
4. Standardize dates to `yyyy-MM-dd`
5. Map bus codes (`AC/NAC/SL/VOL`)
6. Normalize status (`CONFIRMED/PENDING/CANCELLED`)
7. Flag invalid records with reason code
8. Compute derived fields (`amount_per_seat`, `age_category`)
9. Aggregate by route and status
10. Categorize age bucket (`MINOR/ADULT/SENIOR`)

## How to Run

From the project root (`bus-cleaning`):

```bash
mvn -q compile exec:java
```

## What You Will See in Console

- Sequential use-case execution (Use Case 1 → Use Case 10)
- Per-record processing and validation logs
- Rejection reasons for invalid records
- Final use-case pass/fail summary
- Output file locations

## Notes

- This project is Java-only (no Python needed).
- Keep your latest input data in `data/raw_booking.csv` before each run.
